Customer / Coupon Inquiry:

A customer comes in and orders food or drinks. The customer service representative asks if the customer is a member of the loyalty program or would like to join. If so the customer’s number, email, or card number is entered here:

Customer Status:

Using a GetStatus call on a phone number or email will provide you with information about that customer and their current coupons so they can be displayed and redeemed:

Here is an example of the GetStatus call in REST:

Reference for https://api.cert.mercuryloyalty.com/v3_0/Loyalty/GetStatus/?apikey={APIKEY}&apisecret={APISECRET}&customeridentifier={MOBILEOREMAIL}&cardnumber={CARDNUMBER}&customerid={CUSTOMERID}

We can see this customer has coupons available and this can be applied to the ticket by clicking on the coupon. We return useful information like the customer’s name and status in the program in case they have a question.

Coupon \ Info Response:

Now that the coupon has been redeemed using the RedeemCoupon call it will no longer be available in that customers list.

Here is an example of the RedeemCoupon call in REST:

Reference for https://api.cert.mercuryloyalty.com/v3_0/Loyalty/RedeemCoupon/?apikey={APIKEY}&apisecret={APISECRET}&code={CODE}&customeridentifier={MOBILEOREMAIL}&cardnumber={CARDNUMBER}&deviceId={DEVICEID}

Now that the coupon has been redeemed using the RedeemCoupon call it will no longer be available in that customers list.

Nag Screen:

If the customer has not yet entered a number into the system we Nag them at the end of the sale as well.

The amount after discount will be sent at the end of the order using AddCredits and a CloseTransactionWithSku will tell us what he purchased. We use the Close Transaction data to do future marketing.

Here is an example of the AddCredits call in REST:

Reference for https://api.cert.mercuryloyalty.com/v3_0/Loyalty/AddCredits/?apikey={APIKEY}&apisecret={APISECRET}&customeridentifier={MOBILEOREMAIL}&units={UNITS}&description={DESCRIPTION}&employee_id={EMPLOYEE_ID}&station_id={STATION_ID}&ticket_id={TICKET_ID}&revenue={REVENUE}&cardnumber={CARDNUMBER}&language={LANGUAGE}&client={CLIENT}&version={VERSION}&delay={DELAY}&testmode={TESTMODE}&match={MATCH}&deviceId={DEVICEID}

Here is an example of the CloseTransactionWithSku call in REST:
